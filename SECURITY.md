# Security Policy

## Supported Versions

Currently supported versions that receive security updates:

| Version | Status | Support |
|---------|--------|---------|
| 4.1.x   | Active | Receives security patches |
| 4.0.x   | Legacy | No longer supported |
| < 4.0   | Legacy | No longer supported |

## Reporting a Vulnerability

We take security issues seriously. If you discover a vulnerability, please report it responsibly:

**Email**: security@jobpilotai.dev

**Do not** open a public GitHub issue for security vulnerabilities.

## What to Include

When reporting a vulnerability, please provide:

- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Potential impact
- Suggested fix (if you have one)

## Response Timeline

- **48 hours**: Initial acknowledgment of your report
- **7 days**: Security assessment and triage
- **30 days**: Target timeline for patch release

## Vulnerability Scope

### In-Scope
- Code vulnerabilities in JobPilotAI
- Configuration issues
- Dependency vulnerabilities affecting the project

### Out-of-Scope
- Job board Terms of Service violations
- Third-party service issues
- User account security (not project-related)

## Disclosure Process

Once a patch is released, we will:
1. Publish a security advisory
2. Credit the reporter (if desired)
3. Announce the update to users

Thank you for helping keep JobPilotAI secure.
